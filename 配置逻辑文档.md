## 1. 区域参数配置

### 1.1 功能说明
配置不同仓库的大包预估体积、目的港占比、货物类型占比等参数，用于方数预估计算。

### 1.2 字段说明

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| warehouse | string | 仓库（东莞/加工区） | 东莞 |
| package_volume | decimal(12,6) | 大包预估体积（立方米） | 0.124589 |
| kanto_ratio | decimal(5,2) | 关东目的港占比（%） | 45.00 |
| kansai_ratio | decimal(5,2) | 关西目的港占比（%） | 55.00 |
| kanto_normal_ratio | decimal(5,2) | 关东普货占比（%） | 70.00 |
| kanto_special_ratio | decimal(5,2) | 关东特货占比（%） | 30.00 |
| kansai_normal_ratio | decimal(5,2) | 关西普货占比（%） | 60.00 |
| kansai_special_ratio | decimal(5,2) | 关西特货占比（%） | 40.00 |

### 1.3 使用场景
在方数预估时，根据仓库查找对应的区域参数，计算各方数。

### 1.4 注意事项
- 所有占比字段加起来应该等于 100%
- kanto_ratio + kansai_ratio = 100%
- kanto_normal_ratio + kanto_special_ratio = 100%
- kansai_normal_ratio + kansai_special_ratio = 100%

## 2. 航班配置

### 2.1 功能说明
配置不同仓库、不同周几的路由类型（空运/海空）。

### 2.2 字段说明

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| warehouse | string | 仓库（东莞/加工区） | 东莞 |
| weekday | string | 周几（周一~周日） | 周一 |
| kanto_normal | string | 关东普货路由类型（空运/海空） | 空运 |
| kansai_normal | string | 关西普货路由类型（空运/海空） | 海空 |
| kanto_special | string | 关东特货路由类型（空运/海空） | 空运 |
| kansai_special | string | 关西特货路由类型（空运/海空） | 海空 |
| remark | string | 备注 | 周一航班较少 |

### 2.3 使用场景
在方数预估时，根据仓库和周几查找对应的航班配置，确定应配置的空运/海空方数。

### 2.4 注意事项
- 必须为每个仓库、每个周几都配置
- 路由类型只能是"空运"或"海空"

### 3.1 功能说明
配置目的港代码与区域的映射关系。

### 3.2 字段说明

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| port_code | string | 港口代码 | HND |
| region | string | 区域（关东/关西） | 关东 |

### 3.3 使用场景
- 在主单发放时，自动将主单的目的港（dest）映射到区域（关东/关西）

### 3.4 常见港口代码
- 关东：HND（东京羽田）、NRT（东京成田）、TYO（东京）
- 关西：KIX（大阪关西）、OSA（大阪）

## 5. 方数预估

### 5.1 功能说明
根据揽收大包数和区域参数，自动计算总方数、关东/关西总方数、货物分类、应配置方数、实际配置方数、未配置方数。

### 5.2 输入字段

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| collect_date | date | 揽收日期 | 2026-04-06 |
| warehouse | string | 仓库（东莞/加工区/全部） | 加工区 |
| package_count | integer | 揽收大包数 | 231 |
| weight | decimal | 重量（kg） | 4105.65 |
| is_complete | string | 货物袋数是否齐全（是/否） | 是 |

### 5.3 计算逻辑

#### 5.3.1 基础信息

```
总方数 = 揽收大包数 × 大包预估体积

关东总方数 = 总方数 × 关东目的港占比
关西总方数 = 总方数 × 关西目的港占比
```

#### 5.3.2 货物分类

```
关东普货 = 关东总方数 × 关东普货占比
关东特货 = 关东总方数 × 关东特货占比
关西普货 = 关西总方数 × 关西普货占比
关西特货 = 关西总方数 × 关西特货占比
```

#### 5.3.3 应配置方数

根据航班配置（仓库 + 周几）确定路由类型：

```
应配置空运方数 =
  关东普货（如果关东普货路由为空运）+
  关东特货（如果关东特货路由为空运）+
  关西普货（如果关西普货路由为空运）+
  关西特货（如果关西特货路由为空运）

应配置海空方数 =
  关东普货（如果关东普货路由为海空）+
  关东特货（如果关东特货路由为海空）+
  关西普货（如果关西普货路由为海空）+
  关西特货（如果关西特货路由为海空）
```

#### 5.3.4 实际配置方数

**重要：多级回退机制**

```javascript
getEffectiveVolume(order) {
  // 优先级 1：实际方数
  if (order.actual_volume && parseFloat(order.actual_volume) > 0) {
    return parseFloat(order.actual_volume);
  }

  // 优先级 2：打货上限
  if (order.max_volume && parseFloat(order.max_volume) > 0) {
    return parseFloat(order.max_volume);
  }

  // 优先级 3：预估方数
  if (order.est_volume && parseFloat(order.est_volume) > 0) {
    return parseFloat(order.est_volume);
  }

  return 0;
}
```

**空运主单已配置方数**：
```
空运主单已配置方数 = Σ getEffectiveVolume(空运主单)
条件：
1. collect_date === 方数预估的揽收日期
2. warehouse === 方数预估的仓库
3. route_type === '空运'
```

**海空主单已配置方数**：
```
海空主单已配置方数 = Σ getEffectiveVolume(海空主单)
条件：
1. collect_date === 方数预估的揽收日期
2. warehouse === 方数预估的仓库
3. route_type === '海空'
```

#### 5.3.5 未配置方数

```
未配置方数 = 总方数 - 空运主单已配置方数 - 海空主单已配置方数
```

### 5.4 仓库为"全部"时的计算逻辑

当仓库选择"全部"时，系统会汇总所有仓库的数据：

1. **汇总基础数据**：
   - 总揽收大包数 = 东莞大包数 + 加工区大包数
   - 总重量 = 东莞重量 + 加工区重量

2. **汇总计算结果**：
   - 遍历每个仓库（东莞、加工区）
   - 对每个仓库执行单独的计算逻辑
   - 汇总所有仓库的计算结果

3. **汇总实际配置**：
   - 空运主单已配置方数 = 东莞空运主单 + 加工区空运主单
   - 海空主单已配置方数 = 东莞海空主单 + 加工区海空主单

### 5.5 使用场景

1. **查看单个仓库的数据**：选择东莞或加工区
2. **汇总所有仓库的数据**：选择全部
3. **保存预估结果**：点击"保存数据"按钮
4. **编辑历史记录**：在下方表格点击"编辑"按钮

### 5.6 注意事项

- 货物袋数是否齐全为必填项
- 所有数值字段保留 3 位小数（除区域参数的大包预估体积保留 12 位小数）
- 仓库选择"全部"时，自动汇总所有仓库的数据
- 如果当天没有区域参数配置，使用前一天的数据

## 6. 主单发放

### 6.1 功能说明
创建和管理主单，包括揽收信息、航班信息、实际数据等。

### 6.2 输入字段

#### 6.2.1 基础信息

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| collect_date | date | 揽收日期 | 2026-04-06 |
| warehouse | string | 仓库（东莞/加工区） | 加工区 |
| cargo_type | string | 货物属性（普货/特货/混打） | 普货 |
| port | string | 口岸（关东/关西） | 关东 |
| category | string | 类别（自动计算） | 关东普货 |
| status | string | 主单状态（正常/取消/待取消/待定） | 正常 |
| pack_req | string | 打货要求（纸箱/麻袋） | 麻袋 |

#### 6.2.2 打货信息

| 字段名 | 类型 | 说明 | 计算公式 |
|--------|------|------|----------|
| max_volume | string | 打货上限（航线提供） | 手动输入 |
| max_pieces | integer | 打货上限件数（自动计算） | max_volume / 单件体积 |
| est_volume | string | 预估方数（自动计算） | 根据方数预估 |

**单件体积**：
```
东莞-纸箱：0.12 m³
东莞-麻袋：0.159 m³
加工区-纸箱：0.12 m³
加工区-麻袋：0.18 m³
```

#### 6.2.3 航班信息

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| main_no | string | 主单号 | 112-04881612 |
| flight_no | string | 航班号 | MU511 |
| origin | string | 始发机场 | SHA |
| transfer | string | 中转机场 | GMP |
| dest | string | 目的机场 | HND |
| depart_time | string | 起飞时间 | 10:30 |
| arrive_time | string | 到港时间 | 14:20 |
| route_type | string | 路由类型（空运/海空） | 空运 |
| second_flight | string | 二程航班号 | - |
| actual_flight_date | date | 实际起飞日期 | 2026-04-06 |

#### 6.2.4 实际数据

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| actual_pieces | integer | 实际件数 | 53 |
| actual_weight | string | 实际重量（kg） | 1007.466 |
| actual_volume | string | 实际方数 | 8.021 |
| actual_bills | integer | 实际票数 | 1416 |
| issue_card | string | 是否开具售卡（是/否） | 否 |
| remark | string | 备注 | - |

### 6.3 自动计算逻辑

#### 6.3.1 类别（自动计算）
```
类别 = 口岸 + 货物属性
例如：关东 + 普货 = 关东普货
```

#### 6.3.2 打货上限件数（自动计算）
```
打货上限件数 = 打货上限 / 单件体积
（保留整数）
```

#### 6.3.3 预估方数（自动计算）

**查找逻辑**：
1. 先查找当天的方数预估
2. 如果当天没有，查找前一天

**计算逻辑**：
```
if (口岸 === '关东' && 货物属性 === '普货') {
  预估方数 = 关东普货
} else if (口岸 === '关东' && 货物属性 === '特货') {
  预估方数 = 关东特货
} else if (口岸 === '关西' && 货物属性 === '普货') {
  预估方数 = 关西普货
} else if (口岸 === '关西' && 货物属性 === '特货') {
  预估方数 = 关西特货
}
```

#### 6.3.4 航班信息自动填充

**查找逻辑**：
```javascript
匹配条件：
1. flight_no === 航班号
2. origin === 始发机场
3. dest === 目的机场
4. transfer === 中转机场（两边都为空或"-"视为匹配）
```

**填充内容**：
- second_flight（二程航班号）
- depart_time（起飞时间）
- arrive_time（到港时间）

#### 6.3.5 到港日期计算

```
实际到港日期 = 发车日期
if (is_next_day === '是') {
  实际到港日期 = 发车日期 + 1天
}
```

### 6.4 保存逻辑

1. **验证必填字段**：揽收日期、仓库、口岸、货物属性
2. **计算类别、预估方数、打货上限件数**
3. **保存到数据库**
4. **刷新主单列表**

### 6.5 使用场景

1. **新建主单**：填写基础信息、航班信息、实际数据
2. **编辑主单**：在列表点击"编辑"按钮
3. **删除主单**：在列表点击"删除"按钮
4. **筛选主单**：按日期、仓库、口岸、货物属性筛选

### 6.6 注意事项

- 主单号、航班号、始发、目的用于自动填充航班信息
- 路由类型（route_type）必须填写，用于方数预估的实际配置计算
- 实际方数（actual_volume）未填写时，方数预估会使用打货上限（max_volume）
- 重量字段截断保留 2 位小数（不四舍五入）

## 7. 主单查询

### 7.1 功能说明
按条件查询主单，支持导出 Excel。

### 7.2 筛选条件

| 字段名 | 类型 | 说明 |
|--------|------|------|
| collect_date_start | date | 揽收开始日期 |
| collect_date_end | date | 揽收结束日期 |
| actual_flight_date_start | date | 预计起飞开始日期 |
| actual_flight_date_end | date | 预计起飞结束日期 |
| warehouse | string | 仓库（全部/东莞/加工区） |
| origin | string | 始发（全部/具体机场） |
| route_type | string | 路由类型（全部/空运/海空） |
| cargo_type | string | 货物属性（全部/普货/特货） |
| issue_card | string | 是否开具售卡（全部/是/否） |
| main_no | string | 主单号（模糊搜索） |

### 7.3 筛选逻辑

```
主单列表 = 所有主单.filter(主单 => {
  // 揽收日期范围筛选
  if (collect_date_start && collect_date_end) {
    if (collect_date < collect_date_start || collect_date > collect_date_end) return false;
  }

  // 预计起飞日期范围筛选
  if (actual_flight_date_start && actual_flight_date_end) {
    if (actual_flight_date < actual_flight_date_start || actual_flight_date > actual_flight_date_end) return false;
  }

  // 仓库筛选
  if (warehouse !== '全部' && warehouse !== 主单.warehouse) return false;

  // 始发筛选
  if (origin !== '全部' && origin !== 主单.origin) return false;

  // 路由类型筛选
  if (route_type !== '全部' && route_type !== 主单.route_type) return false;

  // 货物属性筛选
  if (cargo_type !== '全部' && cargo_type !== 主单.cargo_type) return false;

  // 是否开具售卡筛选
  if (issue_card === '是' && 主单.issue_card !== '是') return false;
  if (issue_card === '否' && 主单.issue_card !== '否' && 主单.issue_card !== null) return false;

  // 主单号搜索
  if (main_no && !主单.main_no.includes(main_no)) return false;

  return true;
})
```

### 7.4 表格固定列

固定列（滚动时固定在左侧）：
- 揽收日期
- 仓库
- 口岸
- 货物属性
- 路由类型

### 7.5 导出 Excel

**导出字段**：
- 揽收日期、星期、仓库、口岸、货物属性、类别、路由类型
- 主单号、航班号、始发、中转、目的
- 打货上限、预估方数、实际方数、实际件数、实际重量、实际票数
- 状态、打货要求、是否开具售卡
- 实际起飞日期、起飞时间、到港日期、到港时间
- 备注

**文件名格式**：
```
主单查询_YYYY-MM-DD.xlsx
```

### 7.6 使用场景

1. **查询主单**：根据日期、仓库、口岸等条件查询
2. **导出数据**：导出查询结果到 Excel
3. **查看详情**：查看主单的完整信息

### 7.7 注意事项

- 揽收日期范围筛选仅显示日期（不含时分秒）
- 预计起飞日期范围筛选显示完整日期时间（含时分秒）
- 筛选器固定在顶部，表格固定列在左侧
- 是否开具售卡选择"否"时，匹配字段为"否"或 null 的记录

## 8. 欠方余方查询

### 8.1 功能说明
计算预估方数与打货上限的差异，找出欠方（方数不足）和余方（方数过多）的情况。

### 8.2 输入字段

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| collect_date | date | 揽收日期 | 2026-04-06 |
| warehouse | string | 仓库（全部/东莞/加工区） | 全部 |
| port | string | 口岸（全部/关东/关西） | 全部 |
| cargo_type | string | 货物属性（全部/普货/特货） | 全部 |

### 8.3 计算逻辑

#### 8.3.1 查找方数预估

**查找逻辑**：
1. 先查找当天的方数预估
2. 如果当天数据不齐全（is_complete === '否'），查找前一天

#### 8.3.2 计算预估方数

```
if (口岸 === '关东' && 货物属性 === '普货') {
  预估方数 = 关东普货
} else if (口岸 === '关东' && 货物属性 === '特货') {
  预估方数 = 关东特货
} else if (口岸 === '关西' && 货物属性 === '普货') {
  预估方数 = 关西普货
} else if (口岸 === '关西' && 货物属性 === '特货') {
  预估方数 = 关西特货
}
```

#### 8.3.3 计算打货上限汇总

```
打货上限 = Σ 主单.max_volume
条件：
1. collect_date === 揽收日期
2. warehouse === 仓库
3. port === 口岸
4. cargo_type === 货物属性
```

#### 8.3.4 计算欠方/余方

```
差异 = 预估方数 - 打货上限
欠方 = 差异 > 0 ? 差异 : 0
余方 = 差异 < 0 ? Math.abs(差异) : 0
```

### 8.4 查询结果

| 字段名 | 说明 |
|--------|------|
| warehouse | 仓库 |
| port | 口岸 |
| cargo_type | 货物属性 |
| estVolume | 预估方数 |
| maxVolume | 打货上限（汇总） |
| deficit | 欠方（方数不足） |
| surplus | 余方（方数过多） |

### 8.5 使用场景

1. **查看欠方**：预估方数 > 打货上限，说明主单发放不足
2. **查看余方**：预估方数 < 打货上限，说明主单发放过多
3. **调整主单**：根据欠方/余方情况调整主单发放

### 8.6 注意事项

- 如果当天数据不齐全，使用前一天的数据
- 欠方和余方不会同时出现（只有一个有值）
- 支持按仓库、口岸、货物属性筛选

### C. 常见问题

**Q1：为什么实际配置显示为 0？**

A：检查以下几点：
1. 主单是否已保存到数据库（有编辑/删除按钮）
2. 主单的路由类型（route_type）是否填写（空运/海空）
3. 主单的实际方数（actual_volume）或打货上限（max_volume）是否填写

**Q2：未配置方数小于 0 是什么意思？**

A：说明主单发放的方数超过了预估方数，可能存在数据录入错误，需要检查。

**Q3：仓库选择"全部"时怎么计算？**

A：汇总所有仓库的数据，对每个仓库单独计算后汇总结果。

**Q4：到港日期是怎么计算的？**

A：根据发车日期和 is_next_day 字段判断：
- is_next_day === '否'：到港日期 = 发车日期
- is_next_day === '是'：到港日期 = 发车日期 + 1天

**Q5：实际方数、打货上限、预估方数的优先级是什么？**

A：优先级：实际方数 > 打货上限 > 预估方数
- 如果填写了实际方数，使用实际方数
- 否则，如果填写了打货上限，使用打货上限
- 否则，使用预估方数
- 都没填写，使用 0
