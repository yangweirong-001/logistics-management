# 物流管理系统 - 模块逻辑流程文档

> 版本：v1.0
> 更新日期：2026-04-06

---

## 文档说明

本文档说明各个模块之间的数据流和计算逻辑，帮助理解系统运作方式。

## 1. 方数预估模块逻辑

### 1.1 输入数据

```
用户输入：
├── 揽收日期
├── 仓库（东莞/加工区/全部）
├── 揽收大包数
├── 重量
└── 货物袋数是否齐全
```

### 1.2 计算流程

```
【第一步：获取区域参数配置】
│
├─ 查询条件：warehouse = 用户选择的仓库
│
├─ 如果找不到当天的配置：
│   └─ 查找前一天的配置（递归查找）
│
└─ 获取：
    ├── 大包预估体积 (package_volume)
    ├── 关东目的港占比 (kanto_ratio)
    ├── 关西目的港占比 (kansai_ratio)
    ├── 关东普货占比 (kanto_normal_ratio)
    ├── 关东特货占比 (kanto_special_ratio)
    ├── 关西普货占比 (kansai_normal_ratio)
    └── 关西特货占比 (kansai_special_ratio)

【第二步：获取航班配置】
│
├─ 查询条件：
│   ├── warehouse = 用户选择的仓库
│   └─ weekday = 揽收日期对应的星期
│
└─ 获取：
    ├── 关东普货路由类型 (kanto_normal)
    ├── 关西普货路由类型 (kansai_normal)
    ├── 关东特货路由类型 (kanto_special)
    └── 关西特货路由类型 (kansai_special)

【第三步：计算基础信息】
│
├─ 总方数 = 揽收大包数 × 大包预估体积
│
├─ 关东总方数 = 总方数 × 关东目的港占比 / 100
│
└─ 关西总方数 = 总方数 × 关西目的港占比 / 100

【第四步：计算货物分类】
│
├─ 关东普货 = 关东总方数 × 关东普货占比 / 100
│
├─ 关东特货 = 关东总方数 × 关东特货占比 / 100
│
├─ 关西普货 = 关西总方数 × 关西普货占比 / 100
│
└─ 关西特货 = 关西总方数 × 关西特货占比 / 100

【第五步：计算应配置方数】
│
├─ 应配置空运方数 =
│   ├── 如果 关东普货路由 = "空运"：+ 关东普货
│   ├── 如果 关东特货路由 = "空运"：+ 关东特货
│   ├── 如果 关西普货路由 = "空运"：+ 关西普货
│   └── 如果 关西特货路由 = "空运"：+ 关西特货
│
└─ 应配置海空方数 =
    ├── 如果 关东普货路由 = "海空"：+ 关东普货
    ├── 如果 关东特货路由 = "海空"：+ 关东特货
    ├── 如果 关西普货路由 = "海空"：+ 关西普货
    └── 如果 关西特货路由 = "海空"：+ 关西特货

【第六步：获取已保存的主单数据】
│
├─ 查询条件：
│   ├── collect_date = 揽收日期
│   ├── warehouse = 仓库
│   └─ route_type ∈ ["空运", "海空"]
│
├─ 对每个主单，确定有效方数（多级回退）：
│   ├── 优先级1：actual_volume（实际方数，如果 > 0）
│   ├── 优先级2：max_volume（打货上限，如果 > 0）
│   ├── 优先级3：est_volume（预估方数，如果 > 0）
│   └─ 默认：0
│
├─ 空运主单已配置方数 = Σ 空运主单的有效方数
│
└─ 海空主单已配置方数 = Σ 海空主单的有效方数

【第七步：计算实际配置和未配置】
│
├─ 实际配置空运方数 = 空运主单已配置方数
│
├─ 实际配置海空方数 = 海空主单已配置方数
│
└─ 未配置方数 = 总方数 - 实际配置空运方数 - 实际配置海空方数
```

### 1.3 仓库为"全部"时的特殊逻辑

```
当用户选择仓库 = "全部"时：

【第一步：汇总基础数据】
├─ 东莞揽收大包数 + 加工区揽收大包数 = 总揽收大包数
├─ 东莞重量 + 加工区重量 = 总重量
└─ 继续后续计算流程

【第二步：分别计算各仓库数据】
├─ 对仓库 = "东莞" 执行上述完整计算流程
├─ 对仓库 = "加工区" 执行上述完整计算流程
└─ 汇总结果：
    ├── 总方数 = 东莞总方数 + 加工区总方数
    ├── 关东总方数 = 东莞关东 + 加工区关东
    ├── 关西总方数 = 东莞关西 + 加工区关西
    ├── 空运主单已配置方数 = 东莞空运 + 加工区空运
    └── 海空主单已配置方数 = 东莞海空 + 加工区海空
```

### 1.4 数据保存

```
点击"保存数据"按钮：
│
├─ 检查是否已存在同日期同仓库的记录
│   ├── 存在：更新记录
│   └─ 不存在：创建新记录
│
└─ 保存内容：
    ├── collect_date
    ├── weekday
    ├── warehouse
    ├── package_count
    ├── total_volume（总方数）
    ├── kanto_total（关东总方数）
    ├── kansai_total（关西总方数）
    ├── kanto_normal（关东普货）
    ├── kanto_special（关东特货）
    ├── kansai_normal（关西普货）
    ├── kansai_special（关西特货）
    ├── air_volume（应配置空运方数）
    ├── sea_air_volume（应配置海空方数）
    ├── weight
    └── is_complete
```

## 2. 主单发放模块逻辑

### 2.1 输入数据

```
用户输入：
├── 【基础信息】
│   ├── 揽收日期
│   ├── 仓库
│   ├── 货物属性（普货/特货/混打）
│   ├── 口岸（关东/关西）
│   ├── 状态（正常/取消/待取消/待定）
│   └── 打货要求（纸箱/麻袋）
│
├── 【打货信息】
│   ├── 打货上限（航线提供）
│   ├── 预估方数（自动计算）
│   └── 预估件数（自动计算）
│
├── 【航班信息】
│   ├── 主单号
│   ├── 航班号
│   ├── 始发机场
│   ├── 中转机场
│   ├── 目的机场
│   ├── 起飞时间（自动填充）
│   ├── 到港时间（自动填充）
│   ├── 路由类型（自动判断）
│   ├── 二程航班（自动填充）
│   └── 实际起飞日期
│
└── 【实际数据】
    ├── 实际件数
    ├── 实际重量
    ├── 实际方数
    ├── 实际票数
    ├── 是否开具售卡
    └── 备注
```

### 2.2 自动计算流程

```
【第一步：自动计算类别】
│
└─ 类别 = 口岸 + 货物属性
    ├── 例1：关东 + 普货 = "关东普货"
    ├── 例2：关西 + 特货 = "关西特货"
    └── 例3：关东 + 混打 = "关东混打"

【第二步：自动计算打货上限件数】
│
├─ 获取单件体积：
│   ├── 东莞-纸箱：0.12 m³
│   ├── 东莞-麻袋：0.159 m³
│   ├── 加工区-纸箱：0.12 m³
│   └── 加工区-麻袋：0.18 m³
│
├─ 打货上限件数 = 打货上限 / 单件体积
│
└─ 向下取整（整数）

【第三步：自动查找预估方数】
│
├─ 查找顺序：
│   1. 查找 collect_date = 揽收日期 的方数预估
│   2. 如果没找到，查找 collect_date = 揽收日期-1 的方数预估
│   3. 递归查找，直到找到或查到7天前
│
├─ 根据类别匹配对应的数值：
│   ├── 如果 类别 = "关东普货"：预估方数 = 关东普货
│   ├── 如果 类别 = "关东特货"：预估方数 = 关东特货
│   ├── 如果 类别 = "关西普货"：预估方数 = 关西普货
│   └── 如果 类别 = "关西特货"：预估方数 = 关西特货
│
└─ 填充到"预估方数"字段

【第四步：自动匹配路由配置】
│
├─ 触发条件：用户填写了 航班号 + 始发机场 + 目的机场
│
├─ 查找航空路由配置：
│   ├── flight_no = 航班号
│   ├── origin = 始发机场
│   ├── dest = 目的机场
│   └─ transfer = 中转机场（两边都为空或"-"视为匹配）
│
└─ 自动填充：
    ├── 起飞时间 (depart_time)
    ├── 到港时间 (arrive_time)
    ├── 二程航班 (second_flight)
    └── 路由类型 (route_type)

【第五步：自动判断到港日期】
│
├─ 触发条件：用户填写了 实际起飞日期
│
├─ 查找航空路由配置的 is_next_day 字段
│
├─ 计算逻辑：
│   ├── 如果 is_next_day = "否"：到港日期 = 实际起飞日期
│   └── 如果 is_next_day = "是"：到港日期 = 实际起飞日期 + 1天
│
└─ 用于显示到港时间（日期 + 时间）
```

### 2.3 目的港映射逻辑

```
主单发放时，需要将 dest（目的机场）映射到 port（口岸）：

【查询目的港配置】
│
├─ 条件：port_code = dest
│
└─ 获取：region（关东/关西）
│
【自动填充】
├─ port = region
│
└─ 类别 = port + 货物属性
```

### 2.4 保存逻辑

```
点击"保存"按钮：
│
├─ 验证必填字段：
│   ├── 揽收日期
│   ├── 仓库
│   ├── 口岸
│   └── 货物属性
│
├─ 自动计算：
│   ├── 类别（自动）
│   ├── 预估方数（自动查找）
│   └── 打货上限件数（自动计算）
│
├─ 插入/更新主单记录
│
└─ 刷新主单列表
```

### 2.5 与其他模块的关系

```
主单发放 ←→ 方数预估
│
├─ 主单发放时：查找方数预估获取预估方数
│
└─ 方数预估时：汇总主单的实际配置方数

主单发放 ←→ 航空路由配置
│
├─ 主单发放时：自动匹配路由配置填充航班信息
│
└─ 航空路由配置：提供航班详细信息和路由类型

主单发放 ←→ 目的港配置
│
├─ 主单发放时：根据 dest 映射到 port
│
└─ 目的港配置：提供港口代码到区域的映射
```

## 3. 欠方余方查询逻辑

### 3.1 输入数据

```
用户输入：
├── 揽收日期
├── 仓库（全部/东莞/加工区）
├── 口岸（全部/关东/关西）
└── 货物属性（全部/普货/特货）
```

### 3.2 计算流程

```
【第一步：查找方数预估】
│
├─ 查找条件：
│   ├── collect_date = 揽收日期
│   └─ warehouse = 仓库
│
├─ 数据完整性检查：
│   ├── 如果找到的记录 is_complete = "否"：
│   │   └─ 继续查找前一天的记录
│   └─ 直到找到 is_complete = "是" 或查到7天前
│
└─ 获取对应的货物分类数值：
    ├── 如果 口岸 = "关东" && 货物属性 = "普货"：预估方数 = 关东普货
    ├── 如果 口岸 = "关东" && 货物属性 = "特货"：预估方数 = 关东特货
    ├── 如果 口岸 = "关西" && 货物属性 = "普货"：预估方数 = 关西普货
    └── 如果 口岸 = "关西" && 货物属性 = "特货"：预估方数 = 关西特货

【第二步：汇总打货上限】
│
├─ 查询主单表：
│   ├── collect_date = 揽收日期
│   ├── warehouse = 仓库
│   ├── port = 口岸
│   └── cargo_type = 货物属性
│
├─ 汇总所有匹配主单的 max_volume
│
└─ 打货上限 = Σ 主单.max_volume

【第三步：计算差异】
│
├─ 差异 = 预估方数 - 打货上限
│
├─ 欠方 = 差异 > 0 ? 差异 : 0
│
└─ 余方 = 差异 < 0 ? Math.abs(差异) : 0

【注意：欠方和余方不会同时出现】

【第四步：生成查询结果】
│
└─ 每个组合生成一条记录：
    ├── warehouse
    ├── port
    ├── cargo_type
    ├── estVolume（预估方数）
    ├── maxVolume（打货上限）
    ├── deficit（欠方）
    └── surplus（余方）
```

### 3.3 筛选逻辑

```
如果 仓库 = "全部"：
└─ 查询所有仓库的数据，分别计算

如果 口岸 = "全部"：
└─ 查询所有口岸的数据，分别计算

如果 货物属性 = "全部"：
└─ 查询所有货物属性的数据，分别计算

组合查询：
├─ 仓库 × 口岸 × 货物属性
└─ 生成所有可能组合的结果
```

### 3.4 与其他模块的关系

```
欠方余方查询 ←→ 方数预估
│
└─ 获取预估方数数据

欠方余方查询 ←→ 主单发放
│
└─ 获取主单的打货上限数据
```

## 4. 主单查询模块逻辑

### 4.1 输入数据

```
筛选条件：
├── 【日期范围】
│   ├── 揽收日期范围（开始日期 - 结束日期）
│   └── 预计起飞日期范围（开始日期 - 结束日期）
│
├── 【筛选项】
│   ├── 仓库（全部/东莞/加工区）
│   ├── 始发（全部/具体机场，从航空路由配置动态获取）
│   ├── 路由类型（全部/空运/海空）
│   ├── 货物属性（全部/普货/特货）
│   ├── 是否开具售卡（全部/是/否）
│   └── 主单号（模糊搜索）
│
└── 【操作按钮】
    ├── 查询
    └── 重置
```

### 4.2 筛选流程

```
【第一步：构建筛选条件】
│
├─ 揽收日期范围筛选：
│   ├── 如果有开始日期和结束日期：
│   │   └─ collect_date ∈ [开始日期, 结束日期]
│   └─ 如果只有开始日期：
│       └─ collect_date >= 开始日期
│
├─ 预计起飞日期范围筛选：
│   └─ actual_flight_date ∈ [开始日期, 结束日期]
│
├─ 仓库筛选：
│   └─ 如果 仓库 != "全部"：warehouse = 仓库
│
├─ 始发筛选：
│   └─ 如果 始发 != "全部"：origin = 始发
│
├─ 路由类型筛选：
│   └─ 如果 路由类型 != "全部"：route_type = 路由类型
│
├─ 货物属性筛选：
│   └─ 如果 货物属性 != "全部"：cargo_type = 货物属性
│
├─ 是否开具售卡筛选：
│   ├── 如果 选择"是"：issue_card = "是"
│   └─ 如果 选择"否"：issue_card ∈ ["否", null]
│
└─ 主单号搜索：
    └─ main_no 包含 搜索关键词（模糊匹配）

【第二步：执行查询】
│
└─ 从主单表中查询符合所有条件的数据

【第三步：显示结果】
│
├─ 表格显示所有匹配的主单
│
├─ 固定列（滚动时固定在左侧）：
│   ├── 揽收日期
│   ├── 仓库
│   ├── 口岸
│   ├── 货物属性
│   └── 路由类型
│
└─ 动态列（可滚动）：
    ├── 主单号
    ├── 航班号
    ├── 目的港
    ├── 打货上限
    ├── 预估方数
    ├── 实际方数
    ├── 实际件数
    ├── 实际重量
    ├── 实际票数
    ├── 状态
    ├── 打货要求
    ├── 是否开具售卡
    ├── 实际起飞日期
    ├── 起飞时间
    ├── 到港时间
    └── 备注
```

### 4.3 导出 Excel 逻辑

```
点击"导出Excel"按钮：
│
├─ 获取当前查询结果
│
├─ 准备导出数据：
│   ├── 揽收日期、星期、仓库、口岸、货物属性、类别、路由类型
│   ├── 主单号、航班号、始发、中转、目的
│   ├── 打货上限、预估方数、实际方数、实际件数、实际重量、实际票数
│   ├── 状态、打货要求、是否开具售卡
│   ├── 实际起飞日期、起飞时间、到港日期、到港时间
│   └── 备注
│
├─ 转换时间格式：
│   ├── 起飞时间：actual_flight_date + depart_time
│   └── 到港时间：根据 is_next_day 判断日期后 + arrive_time
│
└─ 生成 Excel 文件并下载
```

## 5. 航班异常情况记录逻辑

### 5.1 输入数据

```
用户输入：
├── 主单号（必填）
├── 异常原因（必填）
└── 备注（可选）
```

### 5.2 自动填充逻辑

```
点击"保存"按钮时：
│
├─ 验证必填字段：
│   ├── 主单号
│   └── 异常原因
│
├─ 根据主单号查找主单信息：
│   ├── collect_date（发车日期）
│   ├── depart_date（航班日期）
│   ├── flight_no（航班号）
│   ├── main_no（主单号）
│   ├── actual_bills（票数）

│   └── dest（目的港）
│
└─ 自动填充到异常记录中
```

### 5.3 保存逻辑

│
├─ 验证必填字段

├─ 自动填充航班信息（从主单表获取）

## 7. 关键计算公式汇总

### 7.1 方数预估

```
总方数 = 揽收大包数 × 大包预估体积

关东总方数 = 总方数 × 关东目的港占比 / 100

关西总方数 = 总方数 × 关西目的港占比 / 100

关东普货 = 关东总方数 × 关东普货占比 / 100

关东特货 = 关东总方数 × 关东特货占比 / 100

关西普货 = 关西总方数 × 关西普货占比 / 100

关西特货 = 关西总方数 × 关西特货占比 / 100

有效方数 = 实际方数（优先级1）或 打货上限（优先级2）或 预估方数（优先级3）

未配置方数 = 总方数 - 空运主单已配置方数 - 海空主单已配置方数
```

### 7.2 主单发放

```
类别 = 口岸 + 货物属性

打货上限件数 = 打货上限 / 单件体积（向下取整）

到港日期 = 实际起飞日期
          if (is_next_day = "是") {
            到港日期 = 实际起飞日期 + 1天
          }
```

### 7.3 欠方余方查询

```
差异 = 预估方数 - 打货上限

欠方 = 差异 > 0 ? 差异 : 0

余方 = 差异 < 0 ? Math.abs(差异) : 0
```

## 8. 常见场景示例

### 8.1 场景一：新的一天开始，如何进行方数预估？

```
1. 进入"方数预估"模块
2. 输入揽收日期（今天）
3. 输入仓库（例如：东莞）
4. 输入揽收大包数（例如：231）
5. 输入重量
6. 选择货物袋数是否齐全（是/否）
7. 系统自动：
   - 查找区域参数配置（今天或昨天的）
   - 查找航班配置（今天星期几）
   - 计算总方数、关东/关西总方数
   - 计算货物分类
   - 计算应配置空运/海空方数
   - 查询已保存的主单
   - 计算实际配置方数
   - 计算未配置方数
8. 点击"保存数据"按钮
```

### 8.2 场景二：如何创建一个新主单？

```
1. 进入"主单发放"模块
2. 填写基础信息：
   - 揽收日期
   - 仓库
   - 货物属性
   - 口岸（或填写目的机场，系统自动映射）
   - 状态
   - 打货要求
3. 系统自动：
   - 计算类别
   - 查找方数预估，自动填充预估方数
4. 填写打货上限，系统自动计算打货上限件数
5. 填写航班信息：
   - 主单号
   - 航班号
   - 始发机场
   - 目的机场
   - 实际起飞日期
6. 系统自动：
   - 匹配航空路由配置
   - 自动填充起飞时间、到港时间、二程航班、路由类型
7. 填写实际数据：
   - 实际件数
   - 实际重量
   - 实际方数
   - 实际票数
   - 是否开具售卡
8. 点击"保存"按钮
```

### 8.3 场景三：如何查看欠方余方情况？

```
1. 进入"欠方余方查询"模块
2. 输入揽收日期
3. 选择筛选条件：
   - 仓库（可选择全部）
   - 口岸（可选择全部）
   - 货物属性（可选择全部）
4. 点击"查询"按钮
5. 系统自动：
   - 查找方数预估
   - 汇总主单的打货上限
   - 计算差异
   - 生成欠方/余方结果
6. 查看结果：
   - 如果有欠方：说明主单发放不足，需要增加主单
   - 如果有余方：说明主单发放过多，可能需要调整
```

## 9. 数据完整性检查

### 9.1 方数预估前检查

```
□ 区域参数配置是否完整？
  - 是否有东莞和加工区的配置？
  - 是否每天的配置都齐全？

□ 航班配置是否完整？
  - 是否每个仓库、每个周几都有配置？
  - 路由类型是否正确（空运/海空）？
```

### 9.2 主单发放前检查

```
□ 方数预估是否已保存？
  - 对应日期的方数预估是否存在？

□ 目的港配置是否完整？
  - 主单的 dest 是否有对应的 region？

□ 航空路由配置是否完整？
  - 主单的 航班号+始发+目的 是否有匹配的路由？
```

### 9.3 欠方余方查询前检查

```
□ 方数预估是否已保存？
  - 对应日期的方数预估是否存在？
  - is_complete 是否为"是"？

□ 主单发放是否充足？
  - 打货上限是否填写？
```
