# 物流管理系统

## 项目概览

这是一个基于原生HTML/CSS/JavaScript的物流管理系统，用于管理日本物流的方数预估、主单发放等业务。所有数据存储在浏览器的localStorage中。

## 技术栈

- **前端**: 原生HTML + CSS + JavaScript
- **数据存储**: localStorage
- **服务**: Python SimpleHTTP (端口5000)

## 功能模块

### 配置管理
1. **区域参数配置** - 按仓库配置大包预估体积、关东/关西目的港占比、普货/特货占比
2. **航班配置** - 按仓库和周几配置各类型货物的路由（空运/海空）
3. **目的港配置** - 配置目的港代码对应的区域（关东/关西）
4. **航空路由配置** - 配置航班详情（航班号、始发、中转、目的、起飞/落地时间等）

### 业务操作
1. **方数预估** - 输入揽收日期、仓库、大包数，自动计算各方数并保存
2. **主单发放** - 录入主单信息，自动计算预估方数、预估件数、需求航班日期等
3. **主单查询** - 按日期和仓库查询主单记录
4. **欠方余方查询** - 计算预估方数与打货上限的差异

## 核心计算逻辑

### 方数预估计算
- 总方数 = 揽收大包数 × 大包预估体积
- 关东总方数 = 总方数 × 关东目的港占比
- 关西总方数 = 总方数 × 关西目的港占比
- 各类型方数 = 对应总方数 × 对应占比

### 需求航班日期计算
- 东莞仓：空运 = 揽收日期+2天，海空 = 揽收日期+3天
- 加工区：空运 = 揽收日期+1天，海空 = 揽收日期+3天

### 预估件数计算
- 东莞：纸箱 = 预估方数×0.12，麻袋 = 预估方数×0.159
- 加工区：纸箱 = 预估方数×0.12，麻袋 = 预估方数×0.18

## 数据存储结构

所有数据存储在localStorage，使用以下键：
- `logistics_area_config` - 区域参数配置
- `logistics_flight_config` - 航班配置
- `logistics_port_config` - 目的港配置
- `logistics_route_config` - 航空路由配置
- `logistics_volume_estimate` - 方数预估记录
- `logistics_main_orders` - 主单记录

## 开发命令

```bash
# 启动开发服务
python -m http.server 5000 --bind 0.0.0.0
```

## 注意事项

1. 数据存储在浏览器localStorage中，清除浏览器数据会丢失所有配置
2. 方数预估引用的是"前一天"的预估数据
3. 航班配置按"周几"匹配，而非具体日期
